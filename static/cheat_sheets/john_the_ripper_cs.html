John the Ripper — Master Cheat Sheet

Password cracking reference · JtR 1.9+ / Jumbo · All modes & formats

⚡ John the Ripper (JtR) cracks password hashes. Jumbo build adds 100s more formats. Always run against hashes you own or have written authorization to test.

Single Crack Mode
john --single hashes.txtUse login/GECOS info as password candidates

💡 Single mode works best when the hash file includes usernames (e.g. from unshadow). It uses the username, GECOS, home dir as base words and applies mangling rules. Always try this first — it's fast.

Common Mistakes to Avoid

⚠️ Always specify --format= when hash type is ambiguous. JtR auto-detect can pick wrong type and waste time.

⚠️ Don't run full --incremental on bcrypt/yescrypt — impossibly slow. Use wordlist+rules instead.

⚠️ Check ~/.john/john.pot before re-running — already cracked hashes are skipped automatically.

⚠️ Run --show with --format= specified when hashes have multiple types in one file.

💡 Use --fork=N to fully utilize all CPU cores. Match N to your core count for maximum throughput.
