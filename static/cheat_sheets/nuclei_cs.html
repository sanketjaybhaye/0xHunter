Nuclei master cheat sheet — commands, flags, templates, workflows, and YAML reference

Installation & setup
Install
go install -v github.com/projectdiscovery/nuclei/v3/cmd/nuclei@latest
Install via Go (requires Go 1.21+)
brew install nuclei
macOS via Homebrew
docker pull projectdiscovery/nuclei:latest
Docker image
Templates update
nuclei -update-templates
Update official templates to latest
nuclei -update
Update nuclei binary itself
nuclei -ut -ud ~/custom-templates
Update templates to a custom directory
Essential one-liners
Single target
nuclei -u https://target.com
Scan with all default templates
nuclei -u https://target.com -t exposures/
Run only exposure templates
nuclei -u https://target.com -severity critical,high
Only critical/high severity checks
Multiple targets
nuclei -l targets.txt
Scan list of URLs from file
cat urls.txt | nuclei -t cves/
Pipe targets from stdin
nuclei -l targets.txt -c 50 -rl 200
50 concurrent, 200 req/sec rate limit
Common severity badges
critical high medium low info
Template path: Templates are stored at ~/nuclei-templates/ by default. Point -t to any subdirectory (e.g. cves/2024/) to narrow scope dramatically.
Always get written permission before scanning. Many templates are active and will send real HTTP requests.

Template search & list commands
List & search
nuclei -tl
List all available templates
nuclei -tl -tags cve,rce
List templates matching tags
nuclei -tl -severity critical
List critical-severity templates
nuclei -tl -author pdteam
List templates by specific author
Test & debug templates
nuclei -validate -t my-template.yaml
Validate template syntax
nuclei -u target -t my.yaml -debug
Full request/response debug
nuclei -u target -t my.yaml -v
Verbose template execution
Common scan workflows
Bug bounty quick-start
subfinder -d target.com | httpx | nuclei -t exposures/ -t cves/ -severity critical,high -o results.txt
Enumerate subdomains → probe live hosts → scan exposures and critical CVEs
CVE scanning
nuclei -l hosts.txt -t cves/2024/ -c 50 -rl 100 -o cve-results.json -json
Scan all hosts for 2024 CVEs at controlled rate, JSON output
Default credential audit
nuclei -l targets.txt -t default-logins/ -severity critical,high,medium
Check admin panels for default credentials across common software
Cloud misconfiguration scan
nuclei -l targets.txt -tags aws,gcp,azure,kubernetes,docker -o cloud-misconfigs.txt
Find exposed cloud infrastructure and container misconfigs
SSL/TLS audit
nuclei -l hosts.txt -t ssl/ -t tls/ -p 443
Check for weak ciphers, expired certs, HSTS issues, misconfigs
WordPress audit
nuclei -u https://wp-site.com -tags wordpress -t exposures/cms/wordpress/
Full WordPress-specific template set: plugins, themes, core vulns
Workflow YAML (chained templates)
Example workflow file
id: tech-detect-exploit info: name: Detect then exploit author: me workflows: - template: technologies/apache-detect.yaml matchers: - name: apache subtemplates: - tags: apache,rce
Run exploitation templates only if fingerprinting template matches — reduces noise significantly
Template skeleton
id: example-template-id info: name: Example Vuln Check author: yourhandle severity: high # critical | high | medium | low | info description: | Brief description of what this detects. tags: cve,rce,apache reference: - https://nvd.nist.gov/vuln/detail/CVE-XXXX metadata: cvss-score: 9.8 cve-id: CVE-XXXX-XXXX requests: - method: GET path: - "{{BaseURL}}/vulnerable/endpoint" headers: User-Agent: nuclei-scanner matchers-condition: and matchers: - type: word words: - "vulnerable_string" part: body - type: status status: - 200

Post-processing JSON output
jq recipes
cat results.json | jq '.info.severity'
Extract severity from each finding
cat out.json | jq -r 'select(.info.severity=="critical") | .host'
Get hosts with critical findings only
cat out.json | jq -r '[.host,.matched-at,.["template-id"]] | @csv'
Convert to CSV for reporting
Integration tips
nuclei -sarif-export results.sarif
Import into GitHub Code Scanning, VS Code SARIF viewer
nuclei -markdown-export report/
Generate per-template Markdown docs in directory
OOB / Interactsh
nuclei -u target -interactsh-url https://interactsh.yourserver.com
Use self-hosted interactsh for blind OOB detection
nuclei -u target -ni
-ni disables interactsh (skip OOB templates entirely)
Interactsh is used automatically for templates with oob tag. Set up your own server with interactsh-server for private engagements.
Code protocol templates
code: - engine: - python3 - python source: | import sys print("custom logic here") matchers: - type: word words: - "custom logic here"
Run arbitrary Python/Bash scripts as nuclei templates. Requires -code flag to enable.
nuclei -u target -t my-code.yaml -code
Cloud / DAST mode
nuclei -u target -dast
Enable DAST mode — dynamically discover and test injection points
nuclei -l hosts.txt -cloud-list
List available ProjectDiscovery cloud templates
Scan tuning cheatsheet
Fast / aggressive
nuclei -l t.txt -c 100 -bs 50 -rl 500 -timeout 3
High concurrency, high rate, low timeout — for internal networks
Slow / stealth
nuclei -l t.txt -c 5 -rl 10 -rlm 200 -timeout 15
Low rate, high timeout — for evasion or fragile targets
Integration pipelines
Full recon → scan pipeline
subfinder -d target.com -silent \ | dnsx -silent \ | httpx -silent -threads 100 \ | nuclei \ -t cves/ exposures/ vulnerabilities/ \ -severity critical,high,medium \ -c 30 -rl 150 \ -o findings.json -json \ -stats
Production-ready pipeline: subdomain enum → DNS resolve → HTTP probe → nuclei scan with results saved as JSON
Use -resume to continue interrupted scans. Nuclei auto-creates a resume.cfg file when you hit Ctrl+C.
