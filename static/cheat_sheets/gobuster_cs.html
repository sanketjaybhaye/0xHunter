Gobuster Master Cheat Sheet — complete command reference for directory, DNS, vhost, and S3 scanning

Gobuster Cheat Sheet v3.x
Directory, DNS, VHost, S3 & TFTP enumeration reference
Basic directory/file brute-force
Minimal scan essential
gobuster dir -u http://target.com -w /usr/share/wordlists/dirb/common.txt
Fastest start — common.txt hits ~4k paths.
With extensions essential
gobuster dir -u http://target.com -w common.txt -x php,html,txt,bak,js,json
Appends each extension to every wordlist entry.
Threaded + status filter recommended
gobuster dir -u http://target.com -w medium.txt -t 50 -b 403,404 -x php,html
50 threads, blacklist 403 & 404. Cuts noise significantly.
HTTPS + skip SSL verify
gobuster dir -u https://target.com -w common.txt -k
-k ignores certificate errors (self-signed certs).
Custom headers (auth, bypass)
gobuster dir -u http://target.com -w common.txt -H "Authorization: Bearer TOKEN" -H "X-Custom: val"
Pass multiple -H flags to add any headers per request.
Cookie-based auth
gobuster dir -u http://target.com -w common.txt -c "PHPSESSID=abc123; csrftoken=xyz"
Useful for authenticated scans behind a login wall.
Via proxy (Burp)
gobuster dir -u http://target.com -w common.txt -p http://127.0.0.1:8080
Route all requests through Burp Suite for inspection.
Follow redirects + show full path
gobuster dir -u http://target.com -w medium.txt -r --no-tls-validation
-r follows HTTP redirects; useful for login-redirect chains.
Save output to file
gobuster dir -u http://target.com -w big.txt -o results.txt --no-progress
--no-progress cleans up file output (no spinner noise).
Filter by response size (wordpressed)
gobuster dir -u http://target.com -w common.txt --exclude-length 1234
Exclude a specific response length — great for WAF-redirected 200s.
Username:password basic auth
gobuster dir -u http://target.com -w common.txt -U admin -P password123
HTTP basic authentication. -U user, -P password.
Delay between requests (rate limit evasion)
gobuster dir -u http://target.com -w common.txt --delay 500ms -t 5
Add delay per request + reduce threads to avoid detection.
DNS mode brute-forces subdomains. Requires a valid wordlist of subdomain names (not paths).
Subdomain enumeration
Basic subdomain brute-force essential
gobuster dns -d target.com -w /usr/share/seclists/Discovery/DNS/subdomains-top1million-5000.txt
Finds subdomains via DNS resolution.
Show IP addresses for each found subdomain
gobuster dns -d target.com -w subdomains.txt -i
-i prints resolved IPs alongside found subdomains.
Custom DNS resolver
gobuster dns -d target.com -w subdomains.txt -r 8.8.8.8
-r sets a specific resolver (Google DNS = 8.8.8.8, Cloudflare = 1.1.1.1).
Wildcard detection bypass
gobuster dns -d target.com -w subdomains.txt --wildcard
Forces scan even if wildcard DNS is detected. Filter output manually.
Threaded + output recommended
gobuster dns -d target.com -w subdomains-top1mil.txt -t 50 -o dns-results.txt
50 threads for fast runs, write results to file.
VHost mode manipulates the HTTP Host header to discover virtual hosts on the same IP. Resolves to the target IP directly.
Virtual host enumeration
Basic vhost scan essential
gobuster vhost -u http://target.com -w subdomains.txt
Tests each word as a Host header value against the server.
Append domain to words (gobuster v3.2+) recommended
gobuster vhost -u http://target.com -w wordlist.txt --append-domain
Automatically appends .target.com to each word (word → word.target.com).
Filter by status + exclude length
gobuster vhost -u http://target.com -w wordlist.txt --append-domain -b 302,404 --exclude-length 0
Blacklist redirect & not-found codes; exclude empty responses.
With custom Host header base
gobuster vhost -u http://10.10.10.10 -w wordlist.txt --append-domain -H "Host: FUZZ.target.htb"
Target a raw IP when DNS isn't set up (HackTheBox-style).
S3 bucket enumeration
Basic S3 scan essential
gobuster s3 -w bucket-names.txt
Tests names as S3 bucket identifiers. Checks for public access.
S3 with custom region
gobuster s3 -w bucket-names.txt --region eu-west-1
Google Cloud Storage (GCS)
gobuster gcs -w bucket-names.txt
Tests GCS bucket names. Same flags as s3 mode.
TFTP (UDP file discovery)
gobuster tftp -s target.com -w filenames.txt
Brute-forces filenames on a TFTP server (UDP/69).

Real-world attack recipes
CTF / HTB initial web recon ctf
gobuster dir -u http://target.htb -w /usr/share/seclists/Discovery/Web-Content/directory-list-2.3-medium.txt -x php,html,txt,bak,old,zip,tar.gz -t 40 -b 403,404 -k -o dir-scan.txt
Full initial sweep for CTF web challenges. Good extension coverage.
HTB vhost discovery ctf
gobuster vhost -u http://target.htb -w /usr/share/seclists/Discovery/DNS/subdomains-top1million-5000.txt --append-domain -t 30 -b 302,400,404
Find hidden vhosts on HTB machines. Common pattern in medium/hard boxes.
WordPress endpoint discovery
gobuster dir -u http://wp-site.com -w /usr/share/seclists/Discovery/Web-Content/CMS/wordpress.fuzz.txt -x php,txt -t 30 -b 404
WordPress-specific wordlist for plugin/theme/admin discovery.
API endpoint discovery
gobuster dir -u http://api.target.com -w /usr/share/seclists/Discovery/Web-Content/api/api-endpoints.txt -x json -H "Content-Type: application/json" -H "Accept: application/json" -b 404,405
JSON API recon. Set proper headers so the server serves JSON errors (easier to parse).
Stealthy slow scan (rate limit bypass)
gobuster dir -u http://target.com -w raft-large-directories.txt -t 3 --delay 1s -a "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36" -p http://127.0.0.1:8080
3 threads + 1s delay + browser UA + Burp proxy. Hard to detect/block.
Full subdomain recon pipeline
gobuster dns -d target.com -w subdomains-top1million-20000.txt -i -t 50 -r 1.1.1.1 -o subdomains.txt --no-progress
Fast DNS sweep saving clean output. Feed results into nmap or httpx next.
Backup & config file hunt
gobuster dir -u http://target.com -w /usr/share/seclists/Discovery/Web-Content/raft-large-files.txt -x bak,old,orig,backup,sql,gz,zip,tar,conf,config,cfg,env,log,swp -t 30 -b 404
Hunt for forgotten backups, DB dumps, env files, and config leaks.
Recursive directory scan (shell loop)
# Run gobuster, then re-scan discovered dirs gobuster dir -u http://target.com -w common.txt -t 30 -q | awk '/Status: 200|301/{print $1}' | while read dir; do gobuster dir -u "http://target.com$dir" -w common.txt -t 20; done
Shell one-liner to recursively enumerate each discovered directory.
