Installation
Install
go install -v github.com/projectdiscovery/naabu/v2/cmd/naabu@latest
Go 1.21+ required
brew install naabu
macOS via Homebrew
docker pull projectdiscovery/naabu:latest
Docker image
libpcap (SYN mode requirement)
Required for SYN scanning:
Ubuntu/Debian: sudo apt install libpcap-dev
CentOS/RHEL: sudo yum install libpcap-devel
macOS: brew install libpcap or pre-installed
sudo setcap cap_net_raw=eip $(which naabu)
Grant raw socket capability — run without sudo afterward
Quick Start
Single Target
default scan
$ naabu -host target.com
Top 100 ports. SYN if root, CONNECT if not.
single IP verbose
$ naabu -host 192.168.1.1 -v
Verbose probe tracking
Multiple Targets
from file
$ naabu -list hosts.txt
IPs, CIDRs, hostnames — one per line
stdin pipe
$ cat hosts.txt | naabu -silent
Pipe targets via stdin
CIDR sweep
$ naabu -host 192.168.1.0/24 -top-ports 1000
Entire subnet — top 1000 ports
SYN vs CONNECT Scan
SYN Scan root required
Raw TCP packets — half-open handshake. Faster and stealthier. Default when privileged.
CONNECT Scan no root needed
Full TCP handshake via OS sockets. Works in containers, CI/CD, restricted environments.
SYN scan
$ sudo naabu -host target.com -scan-type syn
Fastest discovery — needs elevated privileges
CONNECT scan
$ naabu -host target.com -scan-type connect
No root — works everywhere
CONNECT + verify
$ naabu -host target.com -scan-type connect -verify
Final TCP connect confirms each result — removes false positives

CDN Detection and Bypass
auto CDN bypass
$ naabu -host target.com -exclude-cdn
If target resolves to Cloudflare, Akamai, Fastly etc. — only checks ports 80/443 and skips full scan. Saves massive overhead on BB programs.
verify CDN and log details
$ naabu -host target.com -verify-cdn
Logs which CDN provider was detected — useful for recon notes
Exclude Filters
Exclude Hosts
inline exclusion
$ naabu -list targets.txt -exclude-hosts 192.168.1.1,10.0.0.1
from blacklist file
$ naabu -list targets.txt -exclude-file blacklist.txt
Exclude Ports
full scan minus specific ports
$ naabu -host target.com -p - -exclude-ports 80,443
All 65535 ports but skip 80 and 443
top 1000 minus noisy mail ports
$ naabu -host target.com -top-ports 1000 -ep 25,110,143
Why combine Naabu + Nmap? Naabu is 10-50x faster at port discovery. Find open ports first, then run Nmap only on those for deep service/version/script fingerprinting. Cuts hours to minutes.
Built-in Nmap Trigger
default nmap
$ sudo naabu -host target.com -nmap
Pipes all open ports into Nmap with -sC -sV automatically
custom nmap arguments
$ sudo naabu -host target.com -nmap-cli 'nmap -sV -sC --min-rate 1000 -oN out.txt'
Custom Nmap args — port list is appended automatically. Nmap must be installed.
Manual Pipe
naabu output piped to nmap
$ naabu -host target.com -silent | awk -F: '{print $2}' | tr '\n' ',' | xargs -I{} nmap -p {} -sV target.com
Extract port list from naabu output and pass to nmap -p manually

Pipeline One-Liners

$ subfinder -d target.com -silent | naabu -silent -top-ports 1000 | httpx -silent -title -tech-detect
Subdomain to open port to live HTTP service with tech fingerprinting
naabu to nuclei network templates
$ naabu -list hosts.txt -silent | nuclei -t network/
Feed host:port pairs directly into nuclei network templates
full vulnerability pipeline
$ subfinder -d target.com -silent | dnsx -silent | naabu -silent -top-ports 1000 | httpx -silent | nuclei -t cves/ -o findings.txt
The complete bug bounty automation pipeline

Performance Profiles
Aggressive — Internal / LAN
max speed
$ sudo naabu -list targets.txt -rate 10000 -c 100 -Pn -timeout 500
Skip host discovery, high rate and concurrency. Safe on LAN.
Stealth — Evasive Targets
slow and quiet
$ naabu -host target.com -rate 200 -c 5 -scan-type connect -timeout 2000
Low rate + connect scan — avoids WAF/IPS threshold alerts
External Bug Bounty
responsible external scan
$ naabu -host target.com -rate 500 -retries 2 -exclude-cdn
Conservative rate + auto-skip CDN hosts
Mass Scanning
large scope with resume
$ naabu -list large-cidr.txt -rate 2000 -o results.txt
Saves resume state — rerun with -resume if interrupted
Troubleshooting
SymptomFix
operation not permittedUse -scan-type connect, or sudo / setcap cap_net_raw=eip
Missing open portsIncrease -timeout, lower -rate, raise -retries
Scan too slowRaise -rate and -c within target tolerance
False positivesAdd -verify to confirm with full TCP connect
Hitting CDN / WAFAdd -exclude-cdn to auto-skip CDN IPs
Container / CI failuresUse -scan-type connect — no root or raw sockets needed
Scanning without authorization may violate laws (CFAA) and provider ToS. Always confirm scope.
