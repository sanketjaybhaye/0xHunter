Subfinder cheat sheet — flags, commands, sources, and pipeline examples

Subfinder cheat sheet v2.x

Fast passive subdomain enumeration tool by ProjectDiscovery

Installation
Go install
go install -v github.com/projectdiscovery/subfinder/v2/cmd/subfinder@latest
Docker
docker pull projectdiscovery/subfinder:latest
Homebrew (macOS/Linux)
brew install subfinder
Core usage
subfinder -d example.com Basic single domain enum
subfinder -d example.com -v Verbose — show sources per result
subfinder -d example.com -o out.txt Save results to file
subfinder -d example.com -oJ JSON output (stdout)
subfinder -dL domains.txt Read domains from file
subfinder -d example.com -silent Only print subdomains (clean pipe)
subfinder -d example.com -all Use all available sources
subfinder -d example.com -recursive Recursive on found subdomains
Quick rate & concurrency
subfinder -d example.com -t 10 10 concurrent goroutines
subfinder -d example.com -timeout 30 30s timeout per source
subfinder -d example.com -rl 100 100 requests/min rate limit
subfinder -d example.com -max-time 5 Cap total runtime to 5 min
Pro tip: Always use -silent when piping into other tools (httpx, naabu, nuclei) so only subdomains are emitted to stdout — no banners, no logs.

Output formats
-o subs.txt Plain text, one sub per line
-oJ -o subs.json JSON with host + source fields
-oD ./output/ Directory: one file per domain
-cs Append source to each result
-oI Resolve & append IP to output
-v Verbose: show source per sub live
Filtering & dedup
Remove wildcards
subfinder -d example.com -nW -silent
Strips subdomains that resolve to wildcard IPs
Sort & dedup with anew
subfinder -d example.com -silent | anew subs.txt
Append only new results to file (tomnomnom/anew)
Filter by TLD / string
subfinder -d example.com -silent | grep "\.staging\."
Grep for specific environment patterns
Count unique results
subfinder -d example.com -silent | sort -u | wc -l
Deduplicate and count total unique subdomains
JSON output fields
FieldDescription
hostSubdomain found
inputOriginal target domain
sourceSource that discovered it

ipResolved IP (only with -oI)
JSON tip: Use jq to process JSON output — e.g. subfinder -d target.com -oJ -silent | jq -r '.host' extracts only hostnames from the JSON stream.

Recon pipelines
Live subdomain check httpx
subfinder -d example.com -silent | httpx -silent
Find subdomains → probe which ones serve HTTP/S
Port scan naabu
subfinder -d example.com -silent | naabu -silent
Enumerate subdomains → port-scan all of them
Vuln scan nuclei
subfinder -d example.com -silent | httpx -silent | nuclei -t cves/
Full chain: find subs → alive check → scan with nuclei CVE templates
Screenshot gowitness
subfinder -d example.com -silent | httpx -silent | gowitness file -f -
Screenshot all live web assets for manual review
Multi-domain bulk
subfinder -dL domains.txt -silent -o all_subs.txt -t 20
Enumerate hundreds of domains concurrently, write all to one file
Continuous monitoring with anew
subfinder -d example.com -silent | anew subs_baseline.txt
Run on a schedule — anew prints ONLY newly discovered subdomains each run
Resolve IPs and deduplicate
subfinder -d example.com -silent -oI | awk '{print $2}' | sort -u
Extract unique IPs from subdomain resolution (-oI flag outputs "sub ip")
Docker one-liner
docker run projectdiscovery/subfinder -d example.com -silent
No install needed — run directly from Docker
ProjectDiscovery full stack
recon → alive → tech detect → vuln scan
subfinder -d example.com -silent \ | httpx -silent -title -tech-detect -status-code \ | tee alive.txt \ | nuclei -t exposures/ -t cves/ -silent
One pipeline from zero to CVE detection with tech fingerprinting along the way
Config file location
Linux / macOS
~/.config/subfinder/config.yaml
Provider / API keys
~/.config/subfinder/provider-config.yaml
Generate default config
subfinder -d example.com # runs once to create config
Use custom config
subfinder -d example.com -config /path/to/config.yaml
provider-config.yaml — API key format
shodan:
  - YOUR_SHODAN_API_KEY
censys:
  - YOUR_CENSYS_API_ID:YOUR_CENSYS_SECRET
github:
  - ghp_TOKEN1
  - ghp_TOKEN2
virustotal:
  - YOUR_VT_API_KEY
securitytrails:
  - YOUR_ST_API_KEY
binaryedge:
  - YOUR_BE_API_KEY
config.yaml — global settings
resolvers:
  - 1.1.1.1
  - 8.8.8.8
sources:
  - shodan
  - crtsh
  - censys
exclude-sources:
  - dnsbufferover
all-sources: false
threads: 10
timeout: 30
Multiple GitHub tokens (recommended)
GitHub has aggressive rate limits. Add 3–5 PAT tokens (read-only, no permissions needed) to provider-config.yaml under github: — subfinder round-robins between them automatically, multiplying your effective rate limit.
Env var override: API keys can also be set via environment variables — e.g. export SHODAN_API_KEY=xyz. Useful in CI/CD pipelines where you don't want keys on disk.
