Core Flags Quick Reference
-l, -list <file>Input file with domains (one per line)
-d, -domain <domain>Single domain to query
-r, -resolver <file|ip>Custom resolver(s) or file of resolvers
-t, -threads <n>Concurrent threads (default: 100)
-rl, -rate-limit <n>DNS queries per second (per resolver)
-retries <n>Retry count for failed queries (default: 2)
-timeout <n>DNS query timeout in seconds (default: 5)
-o, -output <file>Output file to write results
-json, -jOutput in JSON format (enriched)
-silentSuppress all output except results
-v, -verboseVerbose mode (show errors, retries)
-statsDisplay stats at the end of run
-respShow resolved value in output
-resp-onlyPrint only resolved values (no domain)
Fundamental Commands
Resolve single domainrecon
$dnsx -d example.com
$dnsx -d example.com -resp
-resp prints: domain [resolved_IP]. Essential for verification.
Resolve from filerecon
$dnsx -l domains.txt -resp -o live.txt
$cat domains.txt | dnsx -resp
JSON enriched outputrecon
$dnsx -l domains.txt -json -o results.json
$dnsx -l domains.txt -json | jq '.host,.a[]'
JSON mode includes: host, resolver, a, aaaa, cname, ns, mx, soa, txt, ptr, timestamp, status_code.
Performance tuningrecon
$dnsx -l domains.txt -t 500 -rl 3000 -retries 3
High-speed: 500 threads, 3000 DNS req/sec. Lower -rl for stealth.

Query Specific Records
A Recordsrecon
$dnsx -l domains.txt -a -resp
IPv4 addresses. Default record type.
AAAA Recordsrecon
$dnsx -l domains.txt -aaaa -resp
IPv6 addresses. Good for IPv6-only infra.
MX Recordsrecon
$dnsx -l domains.txt -mx -resp
Mail server enumeration. Phishing prep.
TXT Recordsrecon
$dnsx -l domains.txt -txt -resp
SPF, DKIM, DMARC, verification tokens.
NS Recordsrecon
$dnsx -l domains.txt -ns -resp
Nameservers. Find zone delegation.
CNAME Recordsrecon
$dnsx -l subdomains.txt -cname -resp
Aliases. Critical for subdomain takeover.
SOA Records — Zone Authorityrecon
$dnsx -l domains.txt -soa -resp
Returns: primary NS, admin email, serial, refresh, retry, expire, TTL. Admin email often leaks internal addresses.
Reverse DNS — PTRrecon
$echo 93.184.216.34 | dnsx -ptr -resp
$cat ips.txt | dnsx -ptr -resp
IP → hostname. Great for discovering internal naming schemes from ASN sweeps.
Zone Transfer — AXFRcritical
$dnsx -d example.com -axfr -resp
Attempts AXFR zone transfer. Dumps ALL records if misconfigured. Always try — still common in internal networks and legacy infra.
Query ALL record types at oncerecon
$dnsx -l domains.txt -a -aaaa -mx -ns -txt -cname -soa -resp -json -o full_dns.json
Full DNS fingerprint of a target. Pipe to jq for parsing.
TXT Record Intel
TXT records leak: cloud providers (v=spf1 include:_spf.google.com), email security posture (v=DMARC1;p=none = no enforcement), third-party SaaS (Zendesk, Salesforce, etc.), and sometimes internal secrets like verification keys.
Subdomain Brute-Forcing with -w
Basic wordlist bruteforcebrute
$dnsx -d example.com -w subdomains.txt
$dnsx -d example.com -w subdomains.txt -resp -o found.txt
Prepends each wordlist entry to domain. Generates and resolves sub.example.com for each word.
Multi-domain bruteforcebrute
$dnsx -l domains.txt -w subdomains.txt -t 200 -o found.txt
Bruteforce each domain in list. Scales across multiple TLDs or root domains.
Recursion: go deeperbrute
$dnsx -d example.com -w sub.txt -resp | awk '{print $1}' | dnsx -w sub.txt -resp
Two-level deep brute: finds dev.api.example.com, stg.auth.example.com etc. Chain more pipes for deeper recursion.

Wildcard detection & bypassbrute
$# dnsx auto-detects and filters wildcard responses
$dnsx -d example.com -w subdomains.txt -wt 5
-wt sets wildcard threshold: number of random probes used to detect wildcard DNS (default: 5). Increase if you see false positives.
Wildcard Domains
If *.example.com resolves, dnsx will still output results. Always cross-verify with HTTP probing (httpx) — many wildcard IPs return 404 or NXDOMAIN at app layer.
Permutation-based discoverybrute
$alterx -l subs.txt | dnsx -resp -o perm_live.txt
alterx (ProjectDiscovery) generates permutations (dev-api, api-prod, etc.) then dnsx resolves them. High signal-to-noise ratio.
Response-Based Filters
Filter by status codefilter
$dnsx -l domains.txt -rc NOERROR
$dnsx -l domains.txt -rc NXDOMAIN -resp
$dnsx -l domains.txt -rc SERVFAIL,REFUSED
Status codes: NOERROR, NXDOMAIN, SERVFAIL, REFUSED, FORMERR, NOTIMPL. NXDOMAIN = dangling domains (takeover candidates!).
Filter by response (exclude IPs/patterns)filter
$dnsx -l domains.txt -a -resp | grep -v '10\.' | grep -v '192\.168'
$dnsx -l domains.txt -a -resp | grep -v '^10\.\|^172\.\|^192\.168'
Strip RFC1918 private IPs. Useful when targeting only external-facing assets.
Output only IPs / resp-onlyfilter
$dnsx -l domains.txt -a -resp-only | sort -u > ips.txt
$dnsx -l domains.txt -cname -resp-only | sort -u
-resp-only prints only the DNS answer (no hostname). Perfect for feeding into nmap, masscan, httpx.
Match / Filter by IP regexfilter
$dnsx -l domains.txt -a -resp | grep '1\.2\.3\.'
$dnsx -l domains.txt -a -json | jq 'select(.a[]? | startswith("1.2.3."))'
Find all subdomains resolving to a specific IP range or CDN block. Great for mapping shared infra.
Subdomain Takeover Detection
Dangling CNAME findercritical
$dnsx -l subs.txt -cname -resp -rc NXDOMAIN
$dnsx -l subs.txt -cname -json | jq 'select(.status_code=="NXDOMAIN") | .host,.cname[]'
CNAME points somewhere + NXDOMAIN = unclaimed third-party service. Check: GitHub Pages, Heroku, Shopify, Fastly, AWS S3, Surge.sh.
Extract specific fields with jqfilter
$dnsx -l d.txt -json | jq -r '[.host, (.a//[]|join(","))] | @tsv'
$dnsx -l d.txt -json | jq -r 'select(.mx != null) | .host'
$dnsx -l d.txt -json | jq -r '.txt[]? // empty'
Resolver Configuration
Use custom resolversstealth
$dnsx -l domains.txt -r 8.8.8.8,1.1.1.1,9.9.9.9
$dnsx -l domains.txt -r resolvers.txt
Don't rely on system resolver for bulk ops. Custom resolvers prevent ISP interference and rate limiting.
Public Resolver Cheatsheet
8.8.8.8 / 8.8.4.4Google DNS — fast, globally cached
1.1.1.1 / 1.0.0.1Cloudflare — fastest (measured), privacy-first
9.9.9.9 / 149.112.112.112Quad9 — blocks malicious domains (may filter)
208.67.222.222OpenDNS — good global coverage
64.6.64.6Verisign — stable, less filtered
185.228.168.9CleanBrowsing — has filtering, avoid for recon
Get valid resolvers list (dnsvalidator)stealth
$dnsvalidator -tL https://public-dns.info/nameservers.txt -threads 100 -o resolvers.txt
$dnsx -l domains.txt -r resolvers.txt -t 300
Validate 2000+ public resolvers first. Use ~500 good ones to massively distribute DNS traffic.
Resolver trust levelsstealth
$# Trusted resolvers: use for auth confirmation (slow but accurate)
$dnsx -l candidates.txt -r 8.8.8.8,1.1.1.1 -t 20
$# Untrusted public resolvers: use for bulk brute (fast but may be poisoned)
$dnsx -l wordlist_out.txt -r resolvers.txt -t 500
Two-Pass Strategy
1) Brute with 500 threads + massive public resolver list (speed). 2) Re-resolve results with 8.8.8.8 + 1.1.1.1 only (accuracy). This removes false positives from poisoned public resolvers.
Powerful Tool Chains
Full recon pipelinechain
$subfinder -d example.com -silent | dnsx -resp -o live_subs.txt
subfinder → passive enumeration. dnsx → resolve to live hosts only. Gold standard for bug bounty recon.
Recon → Portscan → Web probechain
$subfinder -d example.com -silent | \
dnsx -resp -silent | \
httpx -silent -status-code -title -tech-detect
Classic PD stack: subfinder finds → dnsx verifies live → httpx probes HTTP surface.
ASN → IPs → PTR hostnameschain
$asnmap -org "Target Corp" -silent | mapcidr -silent | dnsx -ptr -resp -resp-only
ASN lookup → CIDR expand → reverse DNS. Discovers internal hostnames from IP ranges. Great for infra mapping.
Subdomain takeover pipelinechain
$subfinder -d target.com -silent | \
dnsx -cname -json -silent | \
jq -r 'select(.cname != null) | .host' | \
nuclei -t ~/nuclei-templates/dns/
Find CNAMEs → run nuclei DNS takeover templates. Automated takeover detection pipeline.
Brute + permutations + verifychain
$subfinder -d target.com -silent | \
alterx | \
dnsx -resp -o perms_live.txt
CVE/vuln discovery via IP mappingchain
$dnsx -l subs.txt -a -resp-only -silent | sort -u | naabu -silent | nuclei -silent
Resolve to IPs → port scan → vuln scan. Full attack surface coverage.

OPSEC Considerations
DNS Footprint Warning
High-speed DNS queries (500+ threads, thousands of qps) ARE detectable. DNS logs are standard on enterprise infra. Blue teams use tools like PassiveDNS, Zeek, and Elastic SIEM to correlate unusual DNS patterns.
Slow & low profile querystealth
$dnsx -l domains.txt -t 5 -rl 10 -timeout 8 -retries 1
5 threads, 10 req/sec. Slow but under radar. Use trusted resolvers (1.1.1.1, 8.8.8.8) — queries appear from those IPs, not yours.
Use DoH / DoT (via proxy)stealth
$# Route dnsx through DNS-over-HTTPS proxy
$dnsx -l domains.txt -r 127.0.0.1:5300
Run a local DoH proxy (cloudflared, dnscrypt-proxy) on :5300 and point dnsx at it. Encrypts DNS traffic over HTTPS/TLS to the upstream.
Pro Tips
Stats mode
Add -stats to any run. Shows: total requests, resolved, errors, rate. Essential for long brute runs to monitor progress.
Silent pipeline
Add -silent to suppress status lines. Only resolved domains go to stdout — critical for clean piping to next tool.
Resume runs
dnsx doesn't have built-in resume. Split wordlists with split -l 100000. Or use GNU parallel: parallel -j4 'dnsx -d target.com -w {}'.
De-duplicate IPs
dnsx -resp-only | sort -u gives unique IPs. Feed to nmap -iL. Many subdomains share IPs — scan each IP once.
Forensics & Investigation
Find all domains on same IP (reverse)recon
$echo 1.2.3.4 | dnsx -ptr -resp
$# Combine with shodan or censys for co-hosted domains
SOA admin email harvestrecon
$dnsx -l domains.txt -soa -json | jq -r '.soa.rname // empty' | sort -u
SOA rname field = admin email (dots as separators). Convert first dot to @. Often leaks real admin contacts.
SPF record analysisrecon
$dnsx -l domains.txt -txt -json | jq -r '.txt[]? | select(startswith("v=spf1"))'
Extract SPF records. Reveals: email providers, cloud services, IP allowlists, and third-party vendors used by target.
