Core Commands Essential
Basic Searches
sherlock username
Search one username across all 400+ sites
sherlock user1 user2 user3
Search multiple usernames in a single run
sherlock --input-file users.txt
Load username list from file (one per line)
Site Filtering
sherlock --site github username
Target a single specific site only
sherlock --site github --site twitter username
Target multiple specific sites
Timeout Control
sherlock --timeout 10 username
Set per-site timeout in seconds (default: 60)
sherlock --timeout 1 username
Speed run — fast, may miss slow-responding sites

Complete Flag Reference Reference
Flag Description Example
--help, -h Show help menu and exit sherlock --help
--version Display version number sherlock --version
--verbose, -v Verbose output — displays all site checks including NOT FOUND sherlock -v target
--print-all Print all sites — found and not found sherlock --print-all user
--print-found Print only confirmed found accounts (reduces noise) sherlock --print-found user
--no-color Disable terminal color output (useful for logs/piping) sherlock --no-color user
--timeout N Per-site request timeout in seconds (default: 60) sherlock --timeout 10 user
--site SITE Limit search to specific site(s). Repeatable. sherlock --site reddit user
--proxy URL Route all traffic through a proxy (HTTP/SOCKS) sherlock --proxy http://127.0.0.1:8080 user
--tor Use Tor network via local SOCKS5 proxy on port 9050 sherlock --tor user
--unique-tor Get a new Tor circuit between each site request sherlock --unique-tor user
--csv Save results as CSV file sherlock --csv user
--json FILE Save results as JSON file sherlock --json out.json user
--xlsx Save results as Excel spreadsheet sherlock --xlsx user
--output FILE Specify output filename for text results sherlock --output john.txt john
--folderoutput DIR Write output files to a specific directory sherlock --folderoutput ./case1/ user
--input-file FILE Read usernames from a file (one per line) sherlock --input-file names.txt
--nsfw Include NSFW / adult platform results (disabled by default) sherlock --nsfw user
--local Use locally cached site data (no network fetch for site list) sherlock --local user

OSINT Recon Workflow Methodology
1
Gather usernames & aliases
Collect all known usernames, handles, and email prefixes. Check social profiles, forum posts, WHOIS data.
2
Run quick broad scan
Start with a fast print-found scan to identify active platforms.
sherlock --timeout 10 --print-found username
3
Verify results manually
Visit each returned URL. Sherlock has low false positives but verify profile matches the target's identity.
4
Export & document findings
Save structured output for reporting or further correlation with other OSINT tools.
sherlock --csv --json results.json --folderoutput ./case/ username
5
Pivot on discovered accounts
Use found profiles to gather email, bio info, friend lists, timestamps. Feed into Maltego, Recon-ng, theHarvester.
6
Run username variations
Try common permutations: underscores, dots, numbers appended.
sherlock user_name username1 xusername username_official

OpSec & Anonymity Critical
DO
  • Route scans through Tor or VPN
  • Use --unique-tor for max anonymity
  • Scan from a dedicated OSINT VM
  • Use --timeout to control rate
  • Verify found profiles manually
  • Get written authorization first
  • Document methodology for reports
  • Use disposable research accounts
DON'T
  • Scan targets without authorization
  • Run from your personal IP address
  • Trust results without verification
  • Ignore false positives
  • Store raw results unencrypted
  • Use against private individuals illegally
  • Violate platform terms of service
  • Share target data without consent
🔒
Tor Setup
Start Tor before using --tor: run sudo service tor start on Kali or brew services start tor on macOS. Sherlock auto-connects to SOCKS5 on port 9050.
⚠️
Rate Limiting
Aggressive scanning can trigger rate limits. Use --timeout 30 or higher and consider adding sleep between bulk runs.

Pro Tips & Tricks Intel
🎯
Username Variations
People use patterns. Try john, john_doe, johndoe, j.doe, john.doe99, xjohn, johnreal. Create a wordlist with username-anarchy.
🔗
Chain with Other Tools
Pipe Sherlock output into theHarvester, Maltego, or Recon-ng. Use found profile links for cross-referencing emails and locations.
📋
JSON for Automation
Use --json output to feed results into Python scripts for automated link scraping or correlation with breach databases.
🐍
Python API Usage
Import Sherlock directly: from sherlock_project.sherlock import sherlock for custom automation workflows or integration into your own tools.
⚡
False Positives
Some sites return 200 OK for any username. Always open the returned URLs and verify the profile content matches your target before documenting.
📂
Organized Case Management
Use --folderoutput ./cases/target_name/ to keep each investigation in its own directory. Combine with date stamps: ./cases/johnsmith_2026-06-14/
