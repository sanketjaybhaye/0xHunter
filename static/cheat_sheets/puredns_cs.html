how it works — pipeline stages
Sanitize input
→
massdns (public resolvers)
→
Wildcard detection
→
Validate (trusted resolvers)
Input is lowercased and stripped to valid chars [a-z0-9.-]. massdns does bulk resolution. Wildcard roots are extracted. Trusted resolvers do a final anti-poisoning pass.
core commands
bruteforceBrute-force subdomains from a wordlist
puredns bruteforce wordlist.txt target.com
puredns bruteforce wordlist.txt target.com -r resolvers.txt
puredns bruteforce wordlist.txt target.com -r resolvers.txt --write out.txt
puredns bruteforce wordlist.txt target.com -r resolvers.txt -l 500
Use -q (quiet) to print only valid domains to stdout — ideal for piping.
resolveMass-resolve a list of domains
puredns resolve domains.txt
puredns resolve domains.txt -r resolvers.txt --write resolved.txt
cat domains.txt | puredns resolve -q # from stdin, quiet
Accepts domains from a file or stdin. Filters wildcards and validates against trusted resolvers automatically.
stdinRead from stdin (pipeline-friendly)
cat wordlist.txt | puredns bruteforce - target.com -q
cat domains.txt | puredns resolve -q | tee live.txt
Use - as the wordlist/domain argument to read from stdin. Combined with -q, only valid results go to stdout.
all flags reference
Resolver & Rate Control
-r, --resolvers <file>
Public resolvers list for massdns bulk pass
--resolvers-trusted <file>
Trusted resolvers for final validation pass
-l, --rate-limit <n>
Limit DNS queries/sec to public resolvers
--rate-limit-trusted <n>
Limit queries/sec to trusted resolvers (default ~10/s)
Output & File Control
--write, -w <file>
Save valid resolved domains to file
--write-wildcards <file>
Save detected wildcard roots to file
--write-massdns <file>
Save clean massdns output to file
-q, --quiet
Suppress banner/progress; only valid domains to stdout
Wildcard Control
--skip-wildcard-filter
Skip wildcard detection entirely (faster, noisier)
--wildcard-tests <n>
DNS queries per subdomain to detect all IPs (-n)
--wildcard-batch <n>
Process N subdomains per wildcard batch (e.g. 1000000)
Validation & Processing
--skip-validation
Skip the final trusted-resolver validation step
--skip-sanitize
Skip input lowercasing/character sanitization
--bin <path>
Path to massdns binary (if not in $PATH)
-t, --threads <n>
Number of concurrent threads
bug bounty recon pipeline
1
passive enum (subfinder / amass)
subfinder -d target.com -silent > passive.txt
2
resolve passive results with puredns
puredns resolve passive.txt -r resolvers.txt -q > live_passive.txt
3
active brute-force with wordlist
puredns bruteforce all.txt target.com -r resolvers.txt -q > bruteforced.txt
4
combine & deduplicate
cat live_passive.txt bruteforced.txt | sort -u > all_subs.txt
5
probe for live HTTP(S) targets
cat all_subs.txt | httpx -silent > live_http.txt
advanced usage & combos
memoryLarge wordlist / wildcard batch
puredns bruteforce all.txt target.com -r r.txt --wildcard-batch 1000000
Prevents OOM crashes on huge wordlists with heavy wildcards.
rateRate-limit to avoid bandwidth saturation
puredns bruteforce all.txt target.com -r r.txt -l 500
massdns without a rate limit can saturate your entire connection.
custom binCustom massdns path
puredns bruteforce wl.txt t.com --bin /opt/massdns/bin/massdns
Use when massdns is not on $PATH.
outputSave all output types
puredns bruteforce wl.txt t.com -r r.txt -w valid.txt --write-wildcards wc.txt --write-massdns raw.txt
skip passesSkip steps for speed (less accurate)
puredns resolve domains.txt --skip-wildcard-filter --skip-validation
Trade accuracy for speed. Not recommended for final recon.
load balanceWildcard load balancing detection
puredns bruteforce wl.txt t.com -r r.txt --wildcard-tests 10
Increases test queries to detect wildcard domains with DNS load balancing across many IPs.

pro tips
Always refresh your resolvers list before big scans. Stale public resolvers reduce accuracy and speed. Use dnsvalidator to regenerate.
If puredns crashes mid-scan on a huge wordlist, add --wildcard-batch 1000000 to process in chunks and avoid OOM errors.
massdns runs at unlimited rate by default — always use -l to cap queries/sec if you're on a shared or limited connection.
Use -q (quiet mode) in all pipeline steps so only valid resolved domains flow through to the next tool.
For targets with DNS load balancing, increase --wildcard-tests (e.g. 10–20) to correctly identify all possible wildcard IPs.
Combine with httpx, nuclei, and gau for a full subdomain → live host → vuln pipeline.
quick one-liners
# Fastest bruteforce (no output file)
puredns bruteforce all.txt target.com -r resolvers.txt -q
# Bruteforce with saved output and rate limit
puredns bruteforce all.txt target.com -r resolvers.txt -l 300 -w subs.txt
# Resolve a list + pipe straight into httpx
puredns resolve domains.txt -r resolvers.txt -q | httpx -silent
# Memory-safe on huge wordlists with wildcards
puredns bruteforce massive.txt target.com -r resolvers.txt --wildcard-batch 1000000 -w out.txt
# Custom massdns binary location
puredns bruteforce wl.txt t.com --bin /usr/local/bin/massdns -r resolvers.txt
# Subfinder → puredns → httpx full pipeline
subfinder -d target.com -silent | puredns resolve -q | httpx -silent -o live.txt
