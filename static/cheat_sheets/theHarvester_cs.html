Basic syntax

pattern 
theHarvester -d <domain> -b <source(s)> [options]
examples 
# single source
theHarvester -d example.com -b crtsh

# multiple sources, comma-separated, no spaces
theHarvester -d example.com -b bing,crtsh,hackertarget,urlscan

# every supported source
theHarvester -d example.com -b all -l 500 -f results

Pulled directly from the current __main__.py argument parser, so this matches what -h actually prints today.

flagdescription
-d--domainRequired. Company name or domain to search.
-b--sourceData source(s), comma-separated, or all. See §4.
-l--limitLimit results per source. Default 500.
-S--startStart at result number X (pagination/resume). Default 0.
-f--filenameSave results to a JSON and XML file (give a base name, no extension).
-p--proxiesRoute requests through proxies defined in proxies.yaml.
-s--shodanCross-reference discovered hosts against Shodan (needs Shodan key).
-c--dns-bruteDNS brute-force subdomains with the built-in wordlist.
-w--wordlistCustom wordlist for -a API-endpoint scanning.
-a--api-scanScan discovered hosts for exposed API endpoints.
-r--dns-resolveResolve found subdomains — pass nothing, a comma-separated IP list, or a resolvers file path.
-n--dns-lookupEnable reverse DNS lookups on discovered ranges.
-t--take-overCheck discovered hosts for subdomain takeover risk (dangling CNAMEs etc.).
-q--quietSuppress "missing API key" warnings — cleaner output on unauthenticated-only runs.
--screenshotScreenshot resolved subdomains. Pass an output dir: --screenshot ./shots.
-e--dns-server⚠ Parsed but not currently wired to anything in the codebase — has no real effect. Use -r <file> for custom resolvers instead.
-h--helpShow help and exit.

Practical recon recipes

Fast, free subdomain sweep — no keys needed
theHarvester -d target.com -b crtsh,hackertarget,rapiddns,urlscan,subdomaincenter,subdomainfinderc99 -l 1000 -f target_subdomains
Email harvesting for a social-engineering risk assessment
theHarvester -d target.com -b duckduckgo,yahoo,baidu,hunter,github-code -l 500 -f target_emails
Full-spectrum sweep — resolve hosts, check takeovers, save everything
theHarvester -d target.com -b all -l 1000 -r resolvers.txt -t -f target_full
Quiet run hitting only free sources
theHarvester -d target.com -b crtsh,hackertarget,rapiddns,otx,urlscan -q -f target_quiet
Subdomain → Shodan pivot (ports/banners on discovered hosts)
theHarvester -d target.com -b crtsh,virustotal -s -f target_shodan
Screenshot every resolved subdomain
theHarvester -d target.com -b crtsh,rapiddns -r --screenshot ./screenshots
API endpoint discovery on found hosts
theHarvester -d target.com -b crtsh -a -w api_wordlist.txt
Through a proxy (configure proxies.yaml first)
theHarvester -d target.com -b crtsh,hackertarget -p

Output & chaining into other tools

-f basename writes basename.json and basename.xml to the working directory.

# pull hostnames out of the JSON, dedupe
jq -r '.hosts[]' target_full.json | sort -u > live_hosts.txt

# feed into httpx for live-host probing
cat live_hosts.txt | httpx -silent -title -status-code

# feed into nmap for a quick port sweep
nmap -iL live_hosts.txt -T4 -oA target_nmap

# pull emails for review
jq -r '.emails[]' target_full.json | sort -u

Tips, gotchas & OPSEC

!
Rate limits / CAPTCHAs. Free search-engine sources (Bing, Yahoo, DuckDuckGo) throttle aggressively. Lower -l, space out runs, or rotate through -p proxies.
!
-b all is slow and noisy. It queries every source and warns for every missing key unless you pass -q. For quick recon, hand-pick 5–6 free sources instead.
›
Quoting company names. If you pass a company name instead of a domain (-d "Acme Corp"), wrap it in quotes.
›
-S / --start is for pagination, not a generic offset — only the legacy search-engine modules really honor it.
›
-r with no argument resolves via your system's default resolver. Pass a file or -r 1.1.1.1,8.8.8.8 to use specific resolvers — useful if you want to avoid ISP DNS logging or hit an internal resolver during an engagement.
›
REST API mode. theHarvester also ships a /additional/* REST API. Set THEHARVESTER_API_KEY and send it as the X-API-Key header to wire it into an automation pipeline.
!
Results aren't validated. Certificate-transparency and search-engine sources surface dead or unrelated subdomains. Always pipe through a live-host check (httpx, dnsx) before acting on anything.

Scope & ethics

theHarvester only queries public, third-party data sources — it doesn't touch the target directly, aside from optional DNS resolution and screenshots of hosts it discovers. Still, only run this against domains and organizations you own or have explicit written authorization to test. Aggregating OSINT on a target without permission can cross into unauthorized reconnaissance under most computer-misuse laws.
