Feroxbuster master cheat sheet covering commands, flags, wordlists, and recon workflows

feroxbuster web fuzzer
fast, recursive content discovery written in Rust — master reference
quick-start commands
Basic scan
Minimal — url + wordlist
feroxbuster -u https://target.com -w /usr/share/wordlists/dirb/common.txt
Specify extensions + threads
feroxbuster -u https://target.com -w raft-medium-words.txt -x php,html,js,txt -t 50
With output file
feroxbuster -u https://target.com -w words.txt -o results.txt
Recursive + depth control
Recursive scan, depth 3
feroxbuster -u https://target.com -r --depth 3
Limit discovered dirs (auto-tune)
feroxbuster -u https://target.com --auto-tune --depth 4
Scan from URL list file
feroxbuster --stdin -w words.txt < urls.txt
filtering results
Filter by status code
Only show 200 and 301
-s 200,301
Filter OUT 404, 403
--filter-status 404,403
Common useful set
-s 200,204,301,302,307,401,403
Filter by size / words / lines
Filter response size (bytes)
--filter-size 4321
Filter by word count in response
--filter-words 24
Filter by line count
--filter-lines 10
Filter by regex in response body
--filter-regex "Access Denied|Not Found"
Filter by similarity
Auto-filter similar responses (wildcard detect)
--auto-bail
Manually set similarity threshold (0.0–1.0)
--filter-similar-to https://target.com/404-page
Use --auto-tune + --auto-bail together for noisy targets — feroxbuster will adapt rate and auto-filter wildcard responses.

real-world attack workflows
Initial web recon
Step 1 — fast surface recon
feroxbuster -u https://target.com -w raft-small-words.txt -t 100 -q -o initial.txt
Step 2 — deep scan with extensions
feroxbuster -u https://target.com -w raft-medium-words.txt -x php,js,txt,bak,old,xml --depth 4 -o deep.txt
API endpoint discovery
REST API — common patterns
feroxbuster -u https://api.target.com -w api-endpoints.txt -m GET,POST -H "Content-Type: application/json" -H "Accept: application/json" -s 200,201,400,401,403,404,500
Versioned API paths
feroxbuster -u https://api.target.com/v1 -w api-endpoints.txt --depth 3
Authenticated scan
JWT auth + proxy to Burp
feroxbuster -u https://target.com/dashboard -w raft-medium.txt -H "Authorization: Bearer TOKEN" --proxy http://127.0.0.1:8080 -k
Session cookie
feroxbuster -u https://target.com/admin -b "session=abc123" -w raft-medium.txt
Stealth / low-noise scan
Rate limited + randomised UA
feroxbuster -u https://target.com -w raft-small.txt --rate-limit 10 -t 5 --random-agent
Bypass common WAF headers
-H "X-Forwarded-For: 127.0.0.1" -H "X-Originating-IP: 127.0.0.1"

tips & gotchas
Wildcard detection: if every path returns 200, use --filter-similar-to pointing at a known-404 URL to baseline the noise.
Recursion explosion: always set --depth and --scan-limit on large sites — unlimited recursion can run for hours.
Resume scans: feroxbuster saves a state file automatically. Run with --resume-from ferox-STATE.state to continue after a crash.
Config file: put repeated flags in ~/.config/feroxbuster/ferox-config.toml to avoid typing them every time.
Burp + TLS: always pair --proxy http://127.0.0.1:8080 with -k to ignore Burp's self-signed cert.
JSON output: use --json -o results.json for downstream tooling — fields include url, status, size, words, lines, duration.
