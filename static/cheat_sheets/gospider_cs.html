what is gospider? web crawler
GoSpider is a fast web spider written in Go. It crawls sites, discovers URLs, extracts JS endpoints, finds subdomains, and collects links from sources like Wayback Machine, CommonCrawl, and VirusTotal — essential for recon and bug bounty.
installation
go install
go install github.com/jaeles-project/gospider@latest
from source
git clone https://github.com/jaeles-project/gospider cd gospider && go build -o gospider .
quickstart examples
single site, basic crawl
gospider -s https://target.com -d 2
crawl many sites from file
gospider -S sites.txt -t 20 -c 10
save output to folder
gospider -s https://target.com -o output/ -d 3
extract all URLs quietly
gospider -s https://target.com --no-redirect -q 2>/dev/null
input required (one of)
-s, --site
Single target URL — https://target.com
-S, --sites
File with one URL per line — -S urls.txt
crawl control
-d, --depth
Max crawl depth (default: 1). Use 2-3 for thorough crawl.
-c, --concurrent
Concurrent requests per site (default: 5). Increase for speed.
-t, --threads
Number of parallel sites when using -S (default: 1).
--delay
Delay between requests in ms — --delay 200 for stealth.
--random-delay
Random delay added on top of --delay. Less predictable pattern.
--timeout
Per-request timeout in seconds (default: 10).
--no-redirect
Disable following HTTP redirects.
http / auth
-H, --header
Set custom headers — -H "Authorization: Bearer TOKEN". Repeatable.
--cookie
Send cookies — --cookie "session=abc123"
-u, --user-agent
Custom User-Agent string. Alias -u "Googlebot/2.1"
--proxy
HTTP proxy URL — --proxy http://127.0.0.1:8080 (routes through Burp/MITM)
--blacklist
Regex of URL patterns to skip — --blacklist "logout|delete"
-K, --disable-tls-checks
Ignore TLS/SSL certificate errors.
extra sources passive intel
--other-source
Pull URLs from Wayback, CommonCrawl, VirusTotal & OTX alongside crawling.
--include-subs
Include subdomains in crawl scope (use with --other-source).
--sitemap
Pull and parse sitemap.xml from target — fast endpoint discovery.
--robots
Parse robots.txt for disallowed paths — often reveals hidden dirs.
output / display
-o, --output
Directory to write output files (one file per site).
-q, --quiet
Suppress banner and progress noise; cleaner piping.
--json
Output in JSON format — great for piping to jq.
--no-color
Disable colored terminal output.
output format — raw
[url] https://target.com/login [form] https://target.com/search?q= [js] https://target.com/static/app.js [linkfinder] /api/v1/user/profile [url] https://ext.otherdomain.com/cdn/lib.js
output format — json (--json)
{"input":"https://target.com","source":"https://target.com","type":"url","output":"https://target.com/admin"} {"input":"https://target.com","source":"https://target.com","type":"js","output":"https://target.com/app.js"} {"input":"https://target.com","source":"https://target.com/app.js","type":"linkfinder","output":"/api/v1/token"}
output types / categories
url — discovered hyperlinks
js — JS file URLs
linkfinder — paths extracted from JS
form — HTML form actions
robots/sitemap — passive paths
wayback/cc — archived URLs
piping & grepping output
extract only urls (no external)
gospider -s https://target.com -q | grep "\[url\]" | awk '{print $2}'
extract js endpoints from linkfinder
gospider -s https://target.com -q | grep "\[linkfinder\]" | awk '{print $2}'
get json + jq for all api paths
gospider -s https://target.com --json -q | jq -r '.output' | grep "/api/"
extract unique domains discovered
gospider -s https://target.com -q | grep -oP 'https?://[^/"]+' | sort -u
scope control — stay on target
crawl only target domain (no externals by default)
gospider -s https://target.com -d 2
include all subdomains in scope
gospider -s https://target.com --include-subs -d 2
blacklist logout/delete endpoints (safety)
gospider -s https://target.com --blacklist "logout|signout|delete|remove"
blacklist static assets (faster)
gospider -s https://target.com --blacklist "\.(png|jpg|gif|css|svg|ico|woff)$"
post-processing filters with grep / awk
find API endpoints only
gospider -s https://target.com -q | grep -iE "/api/|/v[0-9]/"
find endpoints with parameters
gospider -s https://target.com -q | grep "?"
find all .js files
gospider -s https://target.com -q | grep "\.js" | awk '{print $2}' | sort -u
find paths with id/token/key params
gospider -s https://target.com -q | grep -iE "id=|token=|key=|secret=|auth="
look for IDOR-prone endpoints
gospider -s https://target.com -q | grep -oP 'https?://[^\s]+' | grep -E "[0-9]{3,}"
deduplicate & sort discovered URLs
gospider -s https://target.com -q \ | grep -oP 'https?://[^\s">]+' \ | sort -u \ | tee urls.txt

workflow 1 — subdomain + crawl everything
# 1. find subdomains subfinder -d target.com -silent | httpx -silent > live.txt # 2. crawl all live hosts gospider -S live.txt -t 20 -c 10 -d 2 -o crawl/ \ --other-source --include-subs -q # 3. extract unique URLs cat crawl/* | grep -oP 'https?://[^\s"]+' | sort -u > all_urls.txt
workflow 2 — authenticated crawl (logged-in user)
gospider -s https://app.target.com/dashboard \ -H "Cookie: session=YOUR_SESSION_COOKIE" \ -H "Authorization: Bearer YOUR_JWT" \ -d 3 -c 5 \ --no-redirect -q | tee auth_urls.txt
workflow 3 — proxy through burp for manual testing
gospider -s https://target.com \ --proxy http://127.0.0.1:8080 \ -K \ -d 2 -c 3 \ --delay 500 --random-delay 300
workflow 4 — js file hunting + secret extraction
# step 1: get all JS URLs gospider -s https://target.com -q \ | grep "\[js\]\|\[linkfinder\]" \ | awk '{print $2}' | sort -u > js_urls.txt # step 2: download JS and scan for secrets cat js_urls.txt | while read url; do curl -sk "$url" done | grep -iE "api_key|secret|password|token|private_key"
workflow 5 — passive only (no active crawl)
# pull from wayback, commoncrawl, virustotal only gospider -s https://target.com \ --other-source \ --include-subs \ -d 0 \ -q | sort -u | tee passive_urls.txt
gospider → ffuf (param fuzzing)
gospider -s https://target.com -q \ | grep "?" | awk '{print $2}' | sort -u \ | while read url; do ffuf -u "${url%=*}=FUZZ" -w /usr/share/seclists/Fuzzing/XSS/xss-payloads.txt -mc 200 done
gospider → nuclei (vuln scan on discovered URLs)
gospider -s https://target.com -q \ | grep-oP 'https?://[^\s"]+' | sort -u \ | nuclei -l - -t ~/nuclei-templates/ -severity medium,high,critical
gospider → sqlmap (auto sql injection)
gospider -s https://target.com -q \ | grep "=" | awk '{print $2}' | sort -u > params.txt sqlmap -m params.txt --batch --level 2 --risk 2
gospider → waybackurls (expand coverage)
# combine live crawl + wayback gospider -s https://target.com -q | awk '{print $2}' > live.txt waybackurls target.com > wayback.txt cat live.txt wayback.txt | sort -u | uro > final.txt
gospider → gf (pattern matching)
# find XSS-prone parameters gospider -s https://target.com -q | awk '{print $2}' | gf xss # find SSRF-prone parameters gospider -s https://target.com -q | awk '{print $2}' | gf ssrf # find open redirect params gospider -s https://target.com -q | awk '{print $2}' | gf redirect
gospider → hakrawler style mass recon
cat targets.txt | while read domain; do echo "[*] Crawling: $domain" gospider -s "https://$domain" -d 2 -c 5 -q \ --other-source --sitemap --robots \ | awk '{print $2}' | sort -u \ >> "$domain-urls.txt" done
stealth & evasion
Use --delay 1000 --random-delay 500 to blend into normal traffic and avoid WAF rate-limiting.
Spoof User-Agent: -u "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36"
Always use -K on internal/staging hosts with self-signed certs.
performance tuning
For many hosts: -t 50 -c 20 gives ~1000 concurrent requests. Combine with -S biglist.txt.
Lower --timeout 5 to skip slow/dead hosts faster during bulk scans.
Use --blacklist "\.(png|jpg|gif|svg|css|woff|ico)$" to skip assets and speed up crawl by 30-50%.
secret hunting in JS
Pipe [js] and [linkfinder] results to SecretFinder or trufflehog to auto-detect leaked API keys, AWS tokens, and private endpoints.
JS endpoints from [linkfinder] are often undocumented API paths not visible from HTML — always test these for IDOR, missing auth, and info disclosure.
robots.txt & sitemap goldmine
Always use --robots --sitemap — robots.txt Disallow entries frequently expose admin panels, backup directories, and internal tools.

legal reminder important
Only use GoSpider on systems you own or have explicit written permission to test. Unauthorized crawling may violate the CFAA, GDPR, and local laws.
