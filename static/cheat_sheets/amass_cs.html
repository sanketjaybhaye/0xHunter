Amass Cheat Sheet

OWASP Amass — Attack Surface Mapping & External Asset Discovery

The amass enum subcommand is the main engine — it performs DNS enumeration and subdomain discovery.

Basic enumeration
amass enum -d example.com
Passive-only enum — no DNS bruteforce. Fast and stealthy.
amass enum -active -d example.com
Active mode — attempts zone transfers, cert grabs, and more.
amass enum -brute -d example.com
Enable DNS bruteforcing with the built-in wordlist.
amass enum -passive -d example.com
Force passive-only (no active DNS queries at all).
Output & scope
amass enum -d example.com -o out.txt
Write discovered subdomains to a file.
amass enum -d example.com -json out.json
Output in JSON format (richer metadata per result).
amass enum -df domains.txt
Enumerate all domains listed in a file.
amass enum -d example.com -dir ./amass-output
Store all output (graph DB, logs) in a custom directory.
Tuning & control
amass enum -d example.com -timeout 30
Set a timeout in minutes (default: no limit).
amass enum -d example.com -max-dns-queries 500
Throttle concurrent DNS queries (lower = stealthier).
amass enum -d example.com -r resolvers.txt
Use a custom list of DNS resolvers.
amass enum -d example.com -nf known.txt
Provide known subdomains as a starting point (alt names feed).
Wordlists & bruteforce
amass enum -brute -w /path/wordlist.txt -d example.com
Bruteforce with a custom wordlist.
amass enum -brute -min-for-recursive 2 -d example.com
Trigger recursive bruteforce after finding N subdomains at a label.
amass enum -d example.com -wm /wordlist.txt
Wordlist for DNS name mutations/alterations.
amass enum -active -alts -d example.com
Generate name alterations and permutations of found subdomains.
Full power — active + brute + config
amass enum -active -brute -alts -d example.com -config config.ini -o results.txt -json results.json -v
Full featured run: active DNS, bruteforce, alterations, API keys from config, verbose output, dual output formats.
amass enum -passive -d example.com -config config.ini -timeout 20 -json passive.json
Quiet passive recon only — OSINT sources via API keys, no DNS noise. Great for stealth assessments.

amass intel discovers root domain names associated with an organization — finding the attack surface perimeter before enumeration.

Org & WHOIS discovery
amass intel -org "Example Corp"
Find domains belonging to an organization by name.
amass intel -whois -d example.com
Use WHOIS data to find related domains (same registrant email, org).
amass intel -whois -d example.com -v
Verbose WHOIS-based pivoting — shows data sources used.
IP / ASN / CIDR
amass intel -ip -d example.com
Show IP addresses alongside discovered domains.
amass intel -asn 15169
Discover domains within a specific ASN (e.g. AS15169 = Google).
amass intel -cidr 192.168.1.0/24
Find domains resolving within a CIDR range (rDNS sweeping).
amass intel -addr 93.184.216.34
Reverse lookup a specific IP address to find hosted domains.
Certificate & active intel
amass intel -active -d example.com
Actively grab TLS certs and extract domain names from SANs.
amass intel -active -cidr 203.0.113.0/24 -p 443,8443
Scan CIDR for TLS certs on specific ports to find domains.
Intel output
amass intel -org "Example Corp" -o root-domains.txt
Save discovered root domains for piping into amass enum.
amass intel -asn 15169 -json asn-intel.json
JSON output from ASN intel for further processing.

amass db queries and manages the local graph database where all results are persisted across runs.

Query & list
amass db -list
List all domains currently stored in the local database.
amass db -d example.com -names
List all subdomain names found for a specific domain.
amass db -d example.com -summary
Show a summary of all data for a domain from the DB.
amass db -d example.com -ip
Show all subdomains with their resolved IP addresses.
Export & compare
amass db -d example.com -json export.json
Export full DB results for a domain in JSON format.
amass db -d example.com -show
Show all sources that discovered each result.
amass db -d example.com -diff
Compare current and previous enum runs to find new/removed assets.
Visualization
amass viz -d3 -d example.com -o graph.html
Generate an interactive D3.js graph of the asset network.
amass viz -dot -d example.com -o graph.dot
Export as DOT format (for Graphviz, Gephi, etc.).
amass viz -maltego -d example.com
Export for import into Maltego for visual link analysis.
DB management
amass db -dir ./amass-output -d example.com -names
Query a custom output directory (non-default DB location).
amass db -names -d example.com | sort -u > unique.txt
Pipe unique subdomains out for further tools (httpx, nuclei, etc.).

Quick reference for the most important flags across all Amass subcommands.

enum — discovery mode
-passiveUse OSINT sources only; no DNS queries sent
-activeEnable active techniques (zone xfer, cert pull)
-bruteDNS bruteforce with wordlist
-altsGenerate subdomain name alterations
-noaltsDisable alterations even in active mode
-norecursiveTurn off recursive bruteforce
-min-for-recursive NMinimum hits before recursing into a label
enum — targeting
-d domainTarget domain (can repeat: -d a.com -d b.com)
-df fileFile containing list of domains
-bl domainBlacklist a subdomain from results
-blf fileBlacklist file with subdomains to exclude
-include-unresolvableKeep results that don't resolve in DNS
enum — DNS & resolvers
-r fileFile of custom resolver IPs
-rf fileFile of trusted resolver IPs (higher confidence)
-max-dns-queries NMax concurrent DNS queries
-dns-qps NDNS queries per second per resolver
-ports 80,443Ports for active banner/cert grabbing
enum — output
-o fileWrite names to output file
-json fileWrite JSON records to file
-oA prefixOutput all formats with a common prefix
-dir pathCustom output/graph DB directory
-vVerbose output
-silentOnly print results (no status/log)
-nf fileKnown names file to seed enumeration
intel flags
-org "name"Search for root domains by org name
-asn NTarget an Autonomous System Number
-cidr x.x.x.x/NTarget a CIDR range
-addr IPReverse-lookup an IP address
-whoisPivot on WHOIS registrant data
-activeActively grab TLS certificates
-p 443,8443Ports to probe for TLS certs
db flags
-listList all enumerated domains

-namesPrint all subdomain names
-ipInclude IPs in output
-summaryShow statistics/summary
-showShow data sources per result
-diffShow changes between enum runs

A structured recon workflow from zero-knowledge to full asset inventory.

Phase 1 — perimeter discovery (intel)
1
Discover root domains by organization
amass intel -org "Target Corp" -config config.ini -o root-domains.txt
2
Pivot via WHOIS registrant data
amass intel -whois -df root-domains.txt -config config.ini -o more-domains.txt
3
Find ASN & CIDR ranges, enumerate them
amass intel -active -asn 12345 -p 443,8443 -o asn-domains.txt
4
Merge all root domains into one master list
cat root-domains.txt more-domains.txt asn-domains.txt | sort -u > all-roots.txt
Phase 2 — subdomain enumeration
5
Passive enum first (stealthy, fast)
amass enum -passive -df all-roots.txt -config config.ini -dir ./output -json passive.json
6
Active enum with bruteforce (targeted, deeper)
amass enum -active -brute -alts -w wordlist.txt -df all-roots.txt -config config.ini -dir ./output -json active.json
7
Feed known subdomains back to deepen discovery
amass db -names -df all-roots.txt | amass enum -passive -nf - -df all-roots.txt -config config.ini
Phase 3 — analysis & handoff
8
Export unique names with IPs from DB
amass db -names -ip -df all-roots.txt | sort -u > final-subs.txt
9
Check live HTTP(S) — pipe into httpx
amass db -names -df all-roots.txt | httpx -silent -o live-hosts.txt
10
Generate visualization for reporting
amass viz -d3 -df all-roots.txt -dir ./output -o graph.html

Amass uses an INI config file (default: ~/.config/amass/config.ini) to manage API keys and tuning. Using API keys dramatically increases result count.

Config file skeleton

Pro tips, common pitfalls, and integration patterns.

Stealth & rate limiting
Tip
Use -passive first. It queries third-party sources, not the target — zero DNS footprint on the target network.
Tip
Lower -max-dns-queries to 200–300 to avoid triggering rate-limit bans from resolvers during brute-force.
Warning
-active mode performs zone transfer attempts and certificate grabs — these are logged server-side and can trigger IDS alerts.
Tool chaining
# Amass → httpx → nuclei pipeline amass db -names -d example.com | httpx -silent | nuclei -t cves/
Full pipeline: subdomains → live check → vulnerability scan.
amass db -names -d example.com | dnsx -a -resp
Resolve subdomains to IPs using dnsx for quick validation.
amass enum -passive -d example.com -silent | tee subs.txt | wc -l
Pipe results to file AND count them simultaneously.
Performance tuning
Tip
Use -timeout 60 for large scopes to prevent indefinitely long runs. Amass can run for hours on large orgs.
Tip
Supply your own resolvers with -r resolvers.txt. Public lists from github.com/trickest/resolvers are regularly validated.
Tip
Use -dir consistently across runs. Amass accumulates results in its graph DB, and repeat runs improve coverage with zero extra effort.
Common mistakes
Pitfall
Not using a config file. Without API keys you're missing 60–70% of potential results. Even 3–4 free API keys make a huge difference.
Pitfall
Ignoring the graph DB. Re-running without -dir loses context between runs. Always use a consistent output directory per target.
Note
Amass v4.x restructured the CLI. Some older flags (like amass enum -ip) moved. Always check amass enum -h for your installed version.
One-liners for bug bounty recon
# Quick passive recon, deduplicated, sorted amass enum -passive -d example.com -config ~/.config/amass/config.ini -silent | sort -u
# Multi-domain org sweep — passive + active + brute amass enum -active -brute -alts -df scope.txt -config config.ini -dir ./$(date +%F) -json results.json
Date-stamped directory keeps each day's run separate for diffing.
# Compare today vs yesterday — find new assets comm -13 <(sort yesterday/subs.txt) <(sort today/subs.txt)
Shell-based diff to surface newly appeared subdomains for immediate investigation.
