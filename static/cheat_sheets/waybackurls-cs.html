Installation

Setup
Method 1 — go install (recommended)
go install github.com/tomnomnom/waybackurls@latest
waybackurls -h

Needs Go 1.17+. Binary lands in $(go env GOPATH)/bin — add it to PATH.

Method 2 — legacy go get
go get -u github.com/tomnomnom/waybackurls
waybackurls example.com

Deprecated for binaries since Go 1.17 — use Method 1 unless you're stuck on an old toolchain.

Method 3 — build from source
git clone https://github.com/tomnomnom/waybackurls.git
cd waybackurls && go build .

Best option if you want to patch the CDX query logic yourself.

Verify install

Run waybackurls -h to confirm it's on PATH. If the shell can't find it, export PATH=$PATH:$(go env GOPATH)/bin.

Optional third source

VirusTotal lookups stay silent unless you export VT_API_KEY=yourkey — without it, you still get Wayback + CommonCrawl.

Core Commands

Essential
Scan a single domain
waybackurls example.com

Includes all subdomains by default.

Bulk scan from a domain list
cat domains.txt | waybackurls > urls.txt

One domain per line on stdin — this is the tool's primary mode.

Restrict to the exact domain
waybackurls -no-subs example.com

Drops every subdomain hit — useful when you've already enumerated subs separately.

Show snapshot dates
waybackurls -dates example.com

Prefixes each URL with its first-seen timestamp, RFC3339 formatted.

Pull archived replay links
cat urls.txt | waybackurls -get-versions

Mode switch — input is treated as full URLs, output is direct archive.org replay links for every snapshot.

Stack flags together
waybackurls -dates -no-subs example.com

Flags compose freely — just keep them before the domain (see Pro Tips).

Quick flag reference
FlagWhat it does
(none)Fetches every known URL for the domain, expanding to all subdomains.
-no-subsKeeps results to the exact domain given — subdomains are filtered out.
-datesAdds the first-seen snapshot timestamp as a column before each URL.
-get-versionsSwitches modes entirely: input becomes specific URLs, output becomes one archive.org replay link per snapshot of each.

Pro Tips & Gotchas

Important
1

Flags must come before the domain

Go's flag parser stops at the first non-flag argument. waybackurls example.com -dates silently ignores -dates. Always write the flag first: waybackurls -dates example.com.

2

Three sources, three reliability tiers

Wayback's CDX index is the primary, fastest source. CommonCrawl is queried in parallel but its index updates less often. VirusTotal only fires if VT_API_KEY is set in your environment — no key means no error, just no third source.

3

Read the -get-versions output format

Each line is a ready-to-open replay URL shaped like web.archive.org/web/{timestamp}if_/{original}. The if_ suffix serves the raw page without the Wayback toolbar — ideal for diffing JS file history snapshot by snapshot.

4

There's no built-in rate limiting

Scanning hundreds of domains back-to-back can get your IP throttled by archive.org. Wrap multi-domain loops with a short sleep between requests.

5

Getting nothing back? Check the basics first

Confirm the domain actually has Wayback history at web.archive.org/web/*/domain.com, and make sure outbound HTTPS to archive.org isn't blocked by a corporate proxy or VPN.
