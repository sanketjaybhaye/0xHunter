Installation & Compilation
Binary Install
sudo apt install masscan
Quick install via apt on Debian/Ubuntu/Kali Linux
brew install masscan
macOS install via Homebrew
Build from Source
git clone https://github.com/robertdavidgraham/masscan
cd masscan && make -j
Compile from source (creates faster binary optimized for your CPU; requires clang or gcc and libpcap-dev)
Fundamental Command Structures
Basic Targets
sudo masscan 192.168.1.0/24 -p80,443,8080 --rate 1000
Scan a subnet for three common web ports at 1,000 packets per second
sudo masscan -iL ranges.txt -p22,445 --rate 5000
Scan multiple targets listed in a text file for SSH and SMB
Sudo & Kernel Requirements
Requires root privileges: Masscan operates outside the operating system's standard TCP/IP network stack to send raw Ethernet frames directly. Therefore, it must be executed using sudo or as the root user.
Scan Speed Warn: Running masscan at rates higher than your network bandwidth can support will cause packet loss, leading to missed open ports. Always size rates to fit your connection.
Interface and Source Configurations
Adapter Binding
sudo masscan 10.0.0.0/8 -p80 -e eth0
Direct scan explicitly through network adapter interface eth0
sudo masscan 10.0.0.0/8 -p80 --adapter-ip 10.0.0.50
Manually assign the source IP address used for outbound scan packets
MAC Address Overrides
sudo masscan 10.0.0.0/8 -p80 --router-mac 00:11:22:33:44:55
Router MAC Override: Crucial when masscan fails to auto-detect the local gateway's MAC address (highly common on virtual networks, AWS, or OpenStack).
sudo masscan 10.0.0.0/8 -p80 --adapter-mac 00:aa:bb:cc:dd:ee
Spoof your adapter's source MAC address for network evasion
Avoiding OS Kernel RST Packet Conflicts
The Reset Packet Conflict: Because masscan runs its own TCP stack, your local operating system doesn't know about connections initiated by masscan. When a target responds with a SYN-ACK, your local OS kernel sees it as unexpected and automatically responds with an RST (reset) packet, closing the connection.

Professional Evasion Fix:
1. Force masscan to use a specific source port: --adapter-port 40000
2. Add a local firewall rule to prevent the OS kernel from responding on that port:
   • Linux: sudo iptables -A INPUT -p tcp --dport 40000 -j DROP
   • macOS: Block port via pf.conf
Port Specification Syntax
TCP Ports
sudo masscan 192.168.1.1 -p80,443,22,8080
Scan a specific comma-separated list of TCP ports
sudo masscan 192.168.1.1 -p1-1000
Scan a contiguous range of ports (1 to 1000)
sudo masscan 192.168.1.1 -p0-65535
Full port scan: Scans all 65,536 ports (note: masscan starts at port 0, not 1)
UDP Port Scanning
sudo masscan 192.168.1.1 -pU:53,U:161,U:69
UDP Port Scanning: Prefix the port list with U: to scan UDP services (DNS, SNMP, TFTP).
sudo masscan 192.168.1.1 -pU:53,t:80
Scan UDP port 53 and TCP port 80 simultaneously
Speed & Rate Limiting
Transmission Rates
sudo masscan 10.0.0.0/8 -p80 --rate 100000
100,000 p/s: The rate at which the entire internet can be scanned for 1 port in under 6 minutes (requires a 1 Gbps fiber uplink).
sudo masscan 192.168.1.0/24 -p1-65535 --rate 10000
Scan all ports for a local subnet at 10,000 p/s (takes about 1.5 minutes)
Pausing and Resuming Scans
Auto-pausing: When you hit Ctrl+C during a masscan execution, it halts the scan and automatically writes its current progression state to a file named paused.conf.
sudo masscan --resume paused.conf
Resume an interrupted scan exactly where it left off using the saved state configuration.
Sharding (Multi-Server Distributed Scans)
Sharding allows you to split a single massive scan across multiple separate servers/instances (e.g. scanning the internet from 3 different VPS instances). Each server scans its own unique, randomized chunk of the target scope.
# Server 1:
sudo masscan 0.0.0.0/0 -p443 --rate 25000 --shard 1/3 -oJ part1.json

# Server 2:
sudo masscan 0.0.0.0/0 -p443 --rate 25000 --shard 2/3 -oJ part2.json

# Server 3:
sudo masscan 0.0.0.0/0 -p443 --rate 25000 --shard 3/3 -oJ part3.json
Use --shard n/m where n is the current node index (starting at 1) and m is the total number of nodes.
Exclude Filters
Exclude Ranges
sudo masscan 0.0.0.0/0 -p80 --exclude 10.0.0.0/8,192.168.0.0/16
Scan all IP space but exclude private network ranges
Exclude Blacklists
sudo masscan 0.0.0.0/0 -p80 --excludefile blacklist.txt
Exclude all IP ranges listed in a blacklist file (e.g. government subnets, DoD, specific ISPs)
Service Banner Grabbing
sudo masscan 192.168.1.0/24 -p80,443,22 --banners --rate 500
Banner Grabbing: Instructs masscan to complete a full TCP handshake once a port is found open to extract banner data (SSH versions, HTTP server titles, SSL certificates, etc.). Must use a low rate (e.g., <1000 p/s) to avoid overwhelming the targets during connections.
Connection Spoofing & Custom Headers
Custom HTTP headers
sudo masscan targets -p80 --banners --http-user-agent "Mozilla/5.0 (Pentest)"
Inject a custom HTTP User-Agent string during HTTP banner retrieval
sudo masscan targets -p80 --banners --http-payload "GET /secret HTTP/1.0\r\n\r\n"
Send a custom HTTP payload string upon establishing a connection
Other service configurations
sudo masscan targets -p443 --banners --hello-ssl
Send an SSL Client Hello packet immediately upon TCP connection to pull certificate details
Masscan Pipelines
sudo masscan 10.0.0.0/8 -p80 --rate 10000 --open-only -oG - | awk '{print $2}'
Grep-friendly Open IPs: Output active IPs directly to stdout, parsing them immediately using awk to generate an IP list.
sudo masscan -iL targets.txt -p80,443 --open-only -oJ masscan.json
Output results to an enriched JSON file containing only active, open ports.
Output Formats & Conversion
Command Line Formats
FlagFormatDescription
-oX <file>XMLStandard XML output (compatible with Nmap parsers)
-oJ <file>JSONClean, readable JSON Lines format
-oG <file>GrepableOld-school Nmap-style grepable text
-oB <file>BinaryExtremely fast binary format (recommended for massive runs)
Binary scan reading and converting
Binary outputs (`-oB`) are much smaller and write significantly faster to disk, preventing I/O bottlenecks. You can read or convert them later.
masscan --readscan results.bin
Read binary scan file and dump contents directly to terminal stdout
masscan --readscan results.bin -oJ results.json
Convert binary output to standard JSON Lines format
